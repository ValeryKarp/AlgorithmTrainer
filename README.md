# Algorithm Trainer

**Algorithm Trainer** — это офлайн-приложение для подготовки к алгоритмическим собеседованиям. Оно предлагает задачи по различным темам (графы, динамическое программирование, жадные алгоритмы и др.), отслеживает прогресс пользователя и адаптивно подбирает задачи на основе слабых мест.

## Основные возможности
-  База алгоритмических задач с подробными разборами.
-  Система рейтингов по темам и общий прогресс (0–100%).
-  Адаптивный подбор задач: чаще предлагаются темы с низким рейтингом.
-  Фильтр по темам: можно выбрать только нужные темы для тренировки.
-  Детальная информация о задаче (сложность, источник).
-  Сброс прогресса (начать заново).

---

## Системные требования
- **Операционная система**: Windows, Linux (macOS также должен работать).
- **Python** 3.8 или новее.
- **Компилятор C++** с поддержкой C++17 (например, MinGW-w64 на Windows, GCC на Linux).
- **CMake** 3.12+.
- **Библиотеки**:
  - pybind11 (устанавливается через pip)
  - PyQt5
  - nlohmann/json (автоматически загружается CMake)

# Сборка и запуск

## 1. Установка зависимостей Python

- Откройте корень проекта в терминале
- Создайте виртуальное окружение
  ```
  python -m venv .venv
  source .venv/bin/activate      # Linux/macOS
  .venv\Scripts\activate         # Windows
  ```
- Установите необходимые пакеты
  ```
  pip install -r requirements.txt
  ```

## 2. Сборка C++ ядра

- Перейдите в директорию **build** (core/build) или создайте, если ее нет
- Создайте файлы для сборки
  ```
  cmake ..
  ```
- Запустите сборку с конфигурацией **Release**
  ```
  cmake --build . --config Release
  ```
- После в core/build/Release появится файл **core.XXX.pyd
- Скопируйте этот файл в корень проекта и переименуйте в core.pyd

## 3. Запуск приложения

- Запустите **main.py** в корне проекта
  ```
  python main.py
  ```

# Добавление задач

| **Поле**    | **Тип** |                                                 **Описание**                                                 |
|:------------|:-------:|:------------------------------------------------------------------------------------------------------------:|
| id          |   int   |                                       Уникальный идентификатор задачи                                        |
| title       | string  |                                               Название задачи                                                |
| topic       | string  |                                            Тема (например графы)                                             |
| difficulty  |   int   |                                             Сложность от 1 до 10                                             |
| question    | string  | Текст задачи. Поддерживает HTML-теги : &lt;b&gt;, &lt;/b&gt;, &lt;i&gt;, &lt;/i&gt; &lt;br&gt;, &lt;code&gt; |
| explanation | string  |                             Разбор решения задачи. Поддерживает те же HTML-теги                              |
| image       | string  |                                 Имя файла изображения к задаче в data/images                                 |
| source      | string  |                          Источник задачи (например: LeetCode 540, Codeforces 330A)                           |

## Пример задачи:
  ``` 
  {
    "id": 1,
    "title": "Парты",
    "topic": "Математика",
    "difficulty": 1,
    "question": "В классе N учеников. Какое минимальное кол-во парт требуется чтобы их рассадить?"
    "explanation": "За партой может сидеть 2 ученика => чтобы рассадить каждого понадобится ⌈N / 2⌉ парт.",
    "image": "image.jpeg",
    "source": "Класическая задача"
  }
  ```
